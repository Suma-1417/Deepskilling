# Exercise: Singleton Pattern

## Sample Output:
LOG: First log entry
LOG: Second log entry
Only one Logger instance exists.

## Concept
The Singleton Pattern ensures only one instance of a class exists in the application.

## Why use it?
- Provides a single point of access.
- Useful for logging, configuration, or database connections.

## Analysis
- Time complexity: O(1) for instance retrieval.
- Ensures consistency across the application.
